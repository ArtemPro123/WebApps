Overview - FTP

FTP was developed in the early 1970s by Abhay Bhushan while he was a student at MIT.

At its most basic, the FTP protocol allows for the transfer of data and files between two computers over an internet connection.

However it was not designed to be a secure protocol, and has many security weaknesses.

More Security - FTP

The need for improved data security over the internet has led to basic FTP being replaced, by alternative file transfer protocols such as FTPS. FTPS is a secure version of the basic FTP protocol, and was created to stop concerns over internet security. FTPS relies on the same basic methodology of the older FTP protocol, but adds a layer of encryption as a security measure to protect data.

Overview - SSH

The Secure Shell protocol has seen steady improvement and increased adoption since 1995. The first version of Secure Shell (SSH1) was designed to replace the non-secure UNIX. Secure Shell version 2 (SSH2), submitted as a draft in 1997. It addressed some of the more serious vulnerabilities and security issues in SSH1 and also provide an improved file transfer solution. Commercially developed applications are now widely available for Windows, UNIX, Linux, MacOS X, and other platforms.
