Technical Details - FTP

FTP is very diffrent form HTTP in a number of ways. First, FTP is stateful, which means that a single connection between the client and the server is established and is maintained during the session.

More Importantly there are 2 distinct types of FTP connections: contol and Data. A control connection is established when the client initiates a session. When the client wishes to transfer, it issues a command to the server requesting for data.

Data connections can be "passive" or "active". Passive connections are used when the client needs to initiate the connection and active connections are used when the server initiates the connection.

Technical Details - SSH

The SSH protocol has an internal architecture with diffrent layers, for example:

The transport layer handles initial key exchange as well as server authentication, and sets up encryption, compression and integrity verification. It exposes to the upper layer an interface for sending and receiving plaintext.

The user authentication layer handles client authentication and provides a number of authentication methods.

The connection layer defines the concept of channels, channel requests and global requests using SSH services. A single SSH connection can host multiple channels simultaneously, each transferring data in both directions.
